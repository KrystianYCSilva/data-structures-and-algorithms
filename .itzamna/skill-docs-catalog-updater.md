# Docs & Catalog Updater

Source: docs/ALGORITHM_CATALOG.md + docs/PROJECT_ROADMAP.md

Description:
How to update the project docs and algorithm catalog after implementations or status changes. Use when: completing an algorithm/DS or changing roadmap.

What it teaches
- Where to record implementation status, API examples, and roadmap changes.

Instructions
1. Update ALGORITHM_CATALOG.md: add algorithm entry, complexity, and status (✅ / ⏳ / 🔸).
2. Update USAGE_EXAMPLES.md with a minimal example demonstrating the API.
3. Update PROJECT_ROADMAP.md if the milestone changes.
4. If tests or API signatures changed, update corresponding docs and examples to match.
5. Commit messages: include doc files changed and brief reason.

Examples
- After adding QuickSelect: add row in ALGORITHM_CATALOG.md and a short example in USAGE_EXAMPLES.md.
